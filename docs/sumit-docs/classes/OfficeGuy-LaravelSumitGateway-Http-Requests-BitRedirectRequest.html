Bit Redirect Request Validation

Validates user redirect from Bit payment page (success/cancel URLs). This is DIFFERENT from BitWebhookRequest (which handles server-to-server IPN).

✅ FIX #13: Validates orderkey when user returns from Bit payment page This prevents users from manually accessing success pages for orders they don't own.

SECURITY: Unlike BitWebhookRequest, this DOES NOT return 200 on validation failure (because this is a browser request, not a SUMIT webhook).

Usage in application routes:

Route::get('/checkout/success', function (BitRedirectRequest $request) {
    $orderId = $request->getOrderId();
    $order = $request->getValidatedOrder();
    return view('checkout.success', compact('order'));
});
